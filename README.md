# PeregrineOps — Public Safety Executive Intelligence System

An AI-powered executive intelligence dashboard for public safety operations. Monitors police, fire, and EMS incidents across San Francisco with real-time anomaly detection and GPT-4o-generated executive briefings.

---

## Overview

PeregrineOps ingests 90 days of real SF Police Department incident data (via Socrata Open Data API), enriches it with synthetic FIRE and EMS incidents, detects operational anomalies using statistical methods, and surfaces insights through an interactive Streamlit dashboard backed by DuckDB and dbt.

---

## Architecture

```
SF Open Data API (Socrata)
        │
        ▼
scripts/ingest_sf_data.py        ← Ingestion + synthetic data + anomaly injection
        │
        ▼
data/peregrine.duckdb            ← Columnar OLAP database
        │
        ▼
dbt_project/                     ← 3-layer data transformation (raw → staging → marts)
        │
        ▼

        │
        ▼
briefing_engine/generator.py     ← OpenAI GPT-4o executive briefing generation
        │
        ▼
dashboard/app.py                 ← Streamlit web dashboard
```

## Features

### Dashboard Tabs
| Tab | Description |
|-----|-------------|
| **Overview** | Daily incident volumes, response time trends, agency breakdown |
| **Anomalies** | Timeline scatter chart and detail table of detected anomalies |
| **Map** | 7-day incident scatter with hotspot overlays |
| **AI Briefing** | Three-paragraph executive situation report generated by GPT-4o |

### KPI Row (live, 24h window)
- Incident counts (24h and 7d)
- Average and P90 response times
- Active and critical anomaly counts

### Anomaly Detection (4 algorithms)

| Algorithm | Method | Threshold |
|-----------|--------|-----------|
| **Volume Spikes** | Rolling 7-day z-score per agency | z ≥ 2.5 |
| **Response Time Spikes** | Rolling 14-day z-score per agency + district | z ≥ 2.0 or P90 > 1800s |
| **Geographic Hotspots** | Grid-cell density z-scores (0.001° ≈ 111m cells) | z ≥ 2.0 |
| **Resource Gaps** | Hourly analysis over last 24h | avg_units < 3 AND volume > 5, or avg_response > 900s |

**Severity classification:**

| Z-Score | Severity |
|---------|----------|
| ≥ 2.5 | MEDIUM |
| ≥ 3.5 | HIGH |
| ≥ 4.5 | CRITICAL |

## Setup

### 1. Clone the repository

```bash
git clone <repo-url>
cd "Public Safety Executive Intelligence System"
```

### 2. Create and activate a virtual environment

```bash
python -m venv .venv
source .venv/bin/activate       # macOS/Linux
.venv\Scripts\activate          # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure environment variables

Create a `.env` file in the project root:

```env

```

> **Warning:** Never commit your `.env` file or API keys to version control.

## Running the Project

### Full pipeline (ingest → transform → test)

```bash
python run_pipeline.py
```

This runs:
1. `scripts/ingest_sf_data.py` — fetches SF data, generates synthetic incidents, writes to DuckDB
2. `dbt run` — transforms raw data through staging into analytics marts
3. `dbt test` — validates data quality

### Dashboard only

```bash
streamlit run dashboard/app.py
```

Then open `http://localhost:8501` in your browser.

### Individual steps

```bash
# Ingestion only
python scripts/ingest_sf_data.py

# dbt transformation only
cd dbt_project && dbt run

# dbt tests only
cd dbt_project && dbt test
```

## Data Flow

1. **Ingest** — Fetches 90 days of real SF Police incident data from the Socrata API. Generates synthetic FIRE and EMS incidents (~30% additional volume). Injects deliberate anomalies (e.g., Tenderloin hotspot). Writes everything to `data/peregrine.duckdb`.

2. **Transform** — dbt models clean, type-cast, and enrich the raw data through three layers:
   - **Raw**: Pass-through views
   - **Staging**: Response tier classification, null handling, type coercion
   - **Marts**: Aggregated analytics ready for the dashboard and anomaly detection

3. **Detect** — `AnomalyDetector` queries the mart tables, runs four statistical algorithms in parallel, and returns ranked anomalies with severity scores.

4. **Brief** — `BriefingGenerator` passes KPI snapshots and top anomalies to GPT-4o, which returns a structured three-paragraph executive situation report.

5. **Visualize** — Streamlit dashboard renders KPIs, trend charts, an interactive incident map, anomaly timeline, and the AI briefing with a 5-minute data cache.
